docker run

-p 端口映射

-e 参数

-d 作为守护进程来运行

–-name 容器名字

–-restart

Docker容器的重启策略如下：

no，默认策略，在容器退出时不重启容器
on-failure，在容器非正常退出时（退出状态非0），才会重启容器
on-failure:3，在容器非正常退出时重启容器，最多重启3次
always，在容器退出时总是重启容器
unless-stopped，在容器退出时总是重启容器，但是不考虑在Docker守护进程启动时就已经停止了的容器

Dockerfile

FROM ubuntu //来自哪个镜像
MAINTAINER lance //作者
RUN sed -i ‘s/archive.ubuntu.com/mirrors.ustc.edu.cn/g’ /etc/apt/sources.list //换源
RUN apt-get update //执行命令
RUN apt-get install -y nginx
COPY index.html /var/www/html //从当前上下文copy文件到镜像指定路径
ENTRYPOINT ["/usr/sbin/nginx", “-g”, “daemon off;”] //入口， 容器启动所执行的命令
EXPOSE 80 //暴露端口
